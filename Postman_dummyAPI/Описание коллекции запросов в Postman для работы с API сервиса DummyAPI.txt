В рамках выполнения задания была разработана коллекция запросов в Postman для тестирования API сервиса dummyapi.io.

Коллекция включает в себя запросы, проверяющие возможность создания, удаления, получения и обновления пользователя 
и постов, а также создания и удаления комментария.

В настройках коллекции на вкладке Authorization выбран тип API Key, где указан токен из регистрации на сайте. 
Это позволяет не передавать ключ явно в каждом запросе и скрывает его при экспорте коллекции.

Базовый URL вынесен в переменную окружения `BASE_URL`. Кроме того, в процессе выполнения запросов с помощью 
скриптов создаются и заполняются динамические переменные: `user_id`, `post_id`, `comm_id`. 
Значения извлекаются из ответов сервера и автоматически подставляются в последующие запросы.

Для обеспечения надежности тестирования в каждый запрос коллекции были добавлены скрипты валидации. 
В частности, реализованы проверки статус-кода ответа, наличие обязательных полей в теле ответа (id),
а также присутствие заголовка Content-Type. 
Итоговое тестирование было выполнено с использованием встроенного средства Postman — Collection Run,
где все запросы коллекции были успешно запущены в заданной последовательности, а добавленные проверки 
подтвердили правильность выполнения всех запросов.
